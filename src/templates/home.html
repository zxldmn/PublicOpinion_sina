WHAT WE DO

Data Crawling

Selected Android Markets have been crawled for Android App data collecting. Scrapy spiders have been used to crawl the market to achieve useful information by parsing the content with xpath.

Auto Categorizing

Intelligent categorizing method has benn applied for auto adjusting grabbed category information, which will be used for auto defining blacklist/whitelist policy of MAM based on category.

Data Analyzing

Charts&&Grids are implied to show the processed data information statistically. APIs are released out step by step to provide the guys who are interested in the data we have.

WHAT PROJECTS WE HAVE

PolySpider

Poly Spider is mainly designed for Android app synchronizing and categorizing by focusing on grabing data from Android Markets. We have done lots of work to make spider automatically and stable, like taking BAN strategy, using Redis as the data&cache store, monitoring spiders by Supervisor and etc.

PolySpiderFrontend

We have separated PolySpider Front-end module from PolySpider project as a standalone one in version 0.3. It focus on data management and analysis in redis server and providing API service in public. Web.py framework are used in this project just cuz it is so simple and easy to use. We may refactory this project by replacing Web.py with Django in the future. PolySpiderFrontend also integrates some interesting services like redis-live, log2web, tor and etc.

PolyTechDocs

PolyTechDocs is a web documents project including all kinds of documents. Jeklly templates are used in this project and it support Markdown, Textile, Liquid, HTML&&CSS. Anyone could pull a request to make contribution.
